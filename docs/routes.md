# Структура URL прототипа

Прототип покрывает роли и сценарии из `yandex.md` (Яндекс.Директ х Telegram CPP) и `ТЗ.md` (анализ кабинета Getblogger). Ниже описана структура маршрутов, которую использует stub‑приложение.

## Общие разделы

| Путь | Назначение |
| --- | --- |
| `/` | Лендинг и выбор роли (рекламодатель/агентство/блогер/админ). |
| `/auth/direct` | Заглушка авторизации через Яндекс.Директ. |
| `/support` | Контакты поддержки и быстрые ссылки на Telegram-ботов. |

## Рекламодатель / Агентство

Агентский функционал строится на тех же экранах, но агент видит селектор клиента.

| Путь | Назначение |
| --- | --- |
| `/advertiser/dashboard` | Сводка активных и архивных кампаний, быстрые метрики (отклики, бюджет, дедлайны). |
| `/advertiser/campaigns` | Таблица кампаний + фильтры (алгоритм Getblogger `/offers`). |
| `/advertiser/campaigns/archive` | Архив кампаний и отчётов (аналог `/offers/archive`). |
| `/advertiser/campaigns/new` | Многошаговый мастер запуска кампании. |
| `/advertiser/campaigns/new/brief` | Шаг 1: бриф, ссылка, требования, бюджет, CPP/CPM (см. 3.1.1 в `yandex.md`). |
| `/advertiser/campaigns/new/audience` | Шаг 2: подбор тематик аудиторий при помощи LLM. |
| `/advertiser/campaigns/new/priority-channels` | Шаг 3: превью каналов, ручные исключения. |
| `/advertiser/campaigns/new/creatives` | Шаг 4: генерация креативов и их редактор. |
| `/advertiser/campaigns/new/launch` | Шаг 5: проверка бюджета, отправка на модерацию. |
| `/advertiser/campaigns/:campaignId/overview` | Статусы, дедлайны, блок расходов. |
| `/advertiser/campaigns/:campaignId/offers` | Все заявки блогеров по кампании, массовые решения (аналог `/offers/detail/{id}`). |
| `/advertiser/campaigns/:campaignId/creatives` | Модерация креативов, комментарии, версии. |
| `/advertiser/campaigns/:campaignId/publications` | Контроль выхода постов, подтверждения публикаций. |
| `/advertiser/campaigns/:campaignId/analytics` | Метрики из `ТЗ.md` (отклики, публикации, охват, CPV). |
| `/advertiser/bot/inbox` | WebView внутри Telegram-бота: все заявки и массовые действия. |
| `/advertiser/settings/notifications` | Политика уведомлений (бот + email). |

## Агентский селектор

| Путь | Назначение |
| --- | --- |
| `/agency/clients` | Список клиентов и быстрые действия. |
| `/agency/clients/:clientId/switch` | Заглушка переключения контекста (видно в шапке на всех `/advertiser/*` путях). |

## Блогер

| Путь | Назначение |
| --- | --- |
| `/blogger/landing` | Публичный лендинг (превью релевантных брифов, CTA «Взять в работу»). |
| `/blogger/onboarding` | Шаги регистрации, ссылка на бот, проверка канала. |
| `/blogger/pricing` | Настройка ставки CPP, предупреждение о рыночной вилке. |
| `/blogger/offers` | Поток входящих брифов (WebView в боте). |
| `/blogger/offers/:offerId` | Детали брифа, варианты креативов, кнопка «Взять в работу». |
| `/blogger/offers/:offerId/creatives` | Передача креативов, история версий с комментариями рекламодателя. |
| `/blogger/publishing` | Текущие публикации, дедлайны, подтверждение факта поста. |
| `/blogger/history` | Архив завершённых интеграций и их выплаты. |
| `/blogger/settings` | Контакты, доступы, уведомления. |

## Telegram-боты и сервисные экраны

| Путь | Назначение |
| --- | --- |
| `/bot/landing` | Выбор ботов и инструкция по авторизации из интерфейса. |
| `/bot/advertiser` | Веб-просмотр заявок рекламодателя (действия Confirm/Reject). |
| `/bot/blogger` | Веб-просмотр брифов для блогера с фильтром по цене. |

## Администратор

| Путь | Назначение |
| --- | --- |
| `/admin/dashboard` | Общие KPI платформы, распределение бюджетов, статус систем. |
| `/admin/users` | Управление пользователями, статусы блогеров (недорег/полный). |
| `/admin/campaigns` | Поток кампаний, статусы модерации, эскалации. |
| `/admin/disputes` | Разрешение спорных ситуаций по публикациям и оплатам. |
| `/admin/settings` | Настройки интеграций и системных лимитов. |

## Дополнительные разделы

| Путь | Назначение |
| --- | --- |
| `/docs/brief-template` | Шаблон брифа и пояснения (требование из CJM). |
| `/docs/faq` | FAQ по модерации и срокам. |
| `/status` | Техническое состояние сервисов (для всех ролей). |

Эта карта служит источником правды для stub‑приложения: все страницы присутствуют как заглушки с кратким описанием сценария и ссылками на соседние шаги.
